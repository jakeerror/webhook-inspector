## ADR-001. Анонимная модель доступа (без аккаунтов)
- **Дата:** 2026-07-06
- **Контекст:** Инструмент для быстрой отладки вебхуков; регистрация — трение.
- **Решение:** Как webhook.site — bin создаётся анонимно, знание `binId` (случайный slug) = доступ. Slug достаточно длинный/случайный, чтобы его нельзя было угадать; bin живёт по TTL.
- **Альтернативы:** JWT/аккаунты — оправдано для командного продукта, но избыточно для v1 и убивает мгновенность.
- **Последствия:** Нет auth-слоя; безопасность держится на неугадываемости slug + TTL + rate limit. Приватные данные в bin жить долго не должны (TTL 7 дней).

## ADR-002. Live-tail через Redis pub/sub (а не только in-process)
- **Дата:** 2026-07-06
- **Контекст:** WebSocket-клиент должен видеть новый запрос сразу; api может масштабироваться в несколько реплик.
- **Решение:** Новый захваченный запрос публикуется в Redis-канал `bin:{id}`; каждый api-инстанс подписан и рассылает событие своим локальным сокетам. Без Redis — рассылка в пределах процесса (graceful degradation, для тестов/локалки).
- **Альтернативы:** только in-process (не работает при >1 реплики); внешний брокер (избыточно).
- **Последствия:** Корректная работа live при горизонтальном масштабировании; в тестах Redis отключается, WS проверяется в рамках процесса.

## ADR-003. SSRF-защита replay — обязательна и тестируется
- **Дата:** 2026-07-06
- **Контекст:** Replay = сервер делает исходящий HTTP-запрос на пользовательский URL → классический SSRF (доступ к внутренней сети, cloud metadata 169.254.169.254).
- **Решение:** Только http/https; резолв хоста в IP и блок приватных/loopback/link-local/ULA диапазонов; запрет редиректов; таймаут; лимит ответа. Нарушение → 403. Покрыто параметризованными тестами.
- **Альтернативы:** доверять пользователю (недопустимо); allowlist доменов (слишком узко для dev-инструмента).
- **Последствия:** Ключевая security-фича проекта; безопасно даже на публичном хостинге.

## ADR-004. TTL и кольцевой лимит запросов
- **Дата:** 2026-07-06
- **Контекст:** Анонимные bins копятся; тело вебхука может быть большим.
- **Решение:** `expires_at` у bin (TTL 7 дней), фоновая asyncio-задача чистит истёкшее. Тело усекается до 256 KB (`body_truncated`). На bin — не более 500 запросов (старые вытесняются).
- **Последствия:** Ограниченный рост БД; предсказуемые ресурсы.

## ADR-005. Применён официальный FastAPI-скилл (agent skill)
- **Дата:** 2026-07-06
- **Контекст:** Через `find-skills` найден и установлен официальный скилл `fastapi/fastapi@fastapi` (источник — репозиторий FastAPI).
- **Решение:** применил его практики к коду: (1) `Annotated`-зависимости с переиспользуемым алиасом `SessionDep`; (2) `response_model` только там, где публичная схема ≠ возвращаемому типу (ORM → Pydantic), иначе возвратный тип; (3) shared-зависимости на уровне роутера (`ingest` rate-limit); (4) `fastapi[standard]` + `[tool.fastapi] entrypoint` и запуск через `fastapi run` в Docker; (5) HTTPX для исходящих запросов (replay).
- **Последствия:** код соответствует актуальным рекомендациям FastAPI; единый источник правды для зависимостей.
